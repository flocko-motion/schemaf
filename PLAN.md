# atlas-base Implementation Plan

## Purpose

`atlas-base` is a project-agnostic framework for building complex services fast.
It provides: Postgres + migrations, nginx reverse proxy, route registry, OpenAPI generation,
TypeScript client generation, and a CLI tool with compose orchestration and codegen commands.

A new project using atlas-base writes **one compose file** and its business logic.
Everything else — DB setup, migrations, health endpoints, OpenAPI, TypeScript client — comes from the framework.

---

## What a project looks like (end goal)

```
my-project/
  compose/
    app.yml              ← x-atlas.project + depends-on framework compose files + project services
  backend/
    main.go              ← ~20 lines: wire db + migrations + api.Serve
    api/
      todos.go           ← init(): api.Register(Route{...}); handlers only
    db/
      queries/
        todos.sql        ← project-specific queries
      query.gen.go       ← committed; generated by `atlas codegen sqlc`
    migrations/
      0001_todos.sql     ← project schema
    Dockerfile
  frontend/
    src/
      main.ts            ← uses api.gen.ts
      api.gen.ts         ← committed; refreshed by `atlas codegen openapi http://localhost:7001`
    Dockerfile
  cli/
    main.go              ← mounts composecmd + project subcommands
    cmd/
      todos.go           ← project CLI commands
```

`main.go` end-state:

```go
func main() {
    basedb.RegisterMigrations(basedb.MigrationSet{Prefix: "todo", Files: migrations})
    conn, _ := basedb.Open(os.Getenv("DATABASE_URL"))
    basedb.RunMigrations(context.Background(), conn)
    log.Fatal(baseapi.Serve(":7001", conn))
}
```

## Port Convention

```
7000      — atlas-base nginx gateway (main entry point for any atlas-base project)
70xx      — atlas-base standard services
  7001    — backend API
  7002    — frontend dev server (Vite)
  7003    — Postgres (internal)

71xx      — atlas-graph
  7110    — Go API
  7111    — Python Core
  7112    — FalkorDB (Redis protocol)
  7113    — FalkorDB Web UI

72xx      — (reserved: atlas-psyche)
73xx      — (reserved: atlas-journal)
...
Each subproject gets its own 7X00 gateway and 7Xxx service range.
```

## Implementation Tasks

### Task 1 — PORTS.md
Update `/r/priv/atlas/PORTS.md` with the port convention above.

### Task 2 — `atlas-base/web/` — nginx gateway image

**`nginx.conf.template`**
```nginx
server {
    listen 80;

    location /api/ {
        proxy_pass ${BACKEND_URL}/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location / {
        proxy_pass ${FRONTEND_URL}/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

Always proxy — no static file mode. `BACKEND_URL` and `FRONTEND_URL` are full `http://...` values.
In dev: `FRONTEND_URL=http://host.docker.internal:7002`. In prod: `FRONTEND_URL=http://myproject-frontend:80`.

**`entrypoint.sh`**
```sh
#!/bin/sh
envsubst '${BACKEND_URL} ${FRONTEND_URL}' \
  < /etc/nginx/templates/nginx.conf.template \
  > /etc/nginx/conf.d/default.conf
exec nginx -g 'daemon off;'
```

**`Dockerfile`**
```dockerfile
FROM nginx:alpine
COPY nginx.conf.template /etc/nginx/templates/
COPY entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh
ENTRYPOINT ["/entrypoint.sh"]
```

### Task 3 — `atlas-base/compose/` — shared framework services

**`postgres.yml`**
- Image: `postgres:17-alpine`
- Container name: `${PROJECT_NAME}-postgres`
- Volume: `${PROJECT_NAME}-postgres-data`
- Network: `${PROJECT_NAME}-net` (creates it)
- Env: `POSTGRES_USER=atlas`, `POSTGRES_PASSWORD=${DB_PASS}`, `POSTGRES_DB=${PROJECT_NAME}`
- Healthcheck: `pg_isready -U atlas`
- x-atlas: `short-name: db`

**`nginx.yml`**
- Build from `../web`
- Container name: `${PROJECT_NAME}-nginx`
- Port: `7000:80`
- Env: `BACKEND_URL=${BACKEND_URL:-http://host.docker.internal:7001}`, `FRONTEND_URL=${FRONTEND_URL:-http://host.docker.internal:7002}`
- `extra_hosts: host.docker.internal:host-gateway`
- Network: `${PROJECT_NAME}-net` (external: true — created by postgres.yml)
- x-atlas: `short-name: nginx`

### Task 4 — `atlas-base/go/db/` — Postgres + migration package

Add `github.com/lib/pq` to `atlas-base/go/go.mod`.

**`db.go`**
```go
func Open(dsn string) (*sql.DB, error)  // opens connection pool, pings
func HealthCheck(db *sql.DB) error
// Pool constants: MaxOpenConns=25, MaxIdleConns=5, ConnMaxLifetime=5min
```

**`migrate.go`**
```go
type MigrationSet struct {
    Prefix string      // e.g. "ab", "todo" — namespaces migrations in ab_migrations table
    Files  embed.FS    // embedded migration SQL files
}

func RegisterMigrations(ms MigrationSet)   // call before RunMigrations
func RunMigrations(ctx context.Context, db *sql.DB) error
// 1. CREATE TABLE IF NOT EXISTS ab_migrations (id SERIAL PRIMARY KEY, prefix TEXT NOT NULL,
//    version INT NOT NULL, name TEXT NOT NULL, applied_at TIMESTAMPTZ DEFAULT NOW(),
//    UNIQUE(prefix, version))
// 2. For each registered MigrationSet in registration order:
//    - Read *.sql files from Files embed.FS, parse version from NNNN_name.sql filename
//    - Query applied versions WHERE prefix = ms.Prefix
//    - Run each unapplied migration in version order; INSERT into ab_migrations after each

func init() {
    RegisterMigrations(MigrationSet{Prefix: "ab", Files: abMigrations})
}
//go:embed migrations/*.sql
var abMigrations embed.FS
```

**Migration prefix model** — each prefix counts from 0001 independently:
```
prefix  | version | name
--------+---------+-------
ab      |    1    | init        ← atlas-base bootstrap
todo    |    1    | todos       ← example project
psyche  |    1    | blackboard  ← future project
```

**`migrations/0001_init.sql`** — creates `ab_migrations` table (idempotent: `CREATE TABLE IF NOT EXISTS`).
`RunMigrations` always creates the table before reading from it, so this migration is self-bootstrapping.

**`queries/migrations.sql`** — sqlc queries used internally:
```sql
-- name: GetAppliedMigrations :many
SELECT version FROM ab_migrations WHERE prefix = $1 ORDER BY version;

-- name: InsertMigration :exec
INSERT INTO ab_migrations (prefix, version, name) VALUES ($1, $2, $3);
```

**Normative sqlc.yaml layout** (generated by `atlas codegen sqlc`, not hand-written):
```yaml
version: "2"
sql:
  - engine: postgresql
    queries: ./queries/
    schema: ./migrations/
    gen:
      go:
        package: db
        out: .
        emit_prepared_queries: false
        emit_interface: false
        emit_exact_table_names: false
```

### Task 5 — `atlas-base/go/api/` — route registry + server

**`registry.go`**
```go
type Route struct {
    Method      string       // "GET", "POST", etc.
    Path        string       // "/api/todos", "/api/todos/{id}"
    Summary     string       // human-readable; appears in OpenAPI
    Handler     http.Handler
    // Optional schema hints for OpenAPI generation:
    ReqType     any          // pointer to request body type (for reflection)
    RespType    any          // pointer to response type (for reflection)
}

var routes []Route

func Register(r Route)
func Routes() []Route
```

**`openapi.go`** — `OpenAPIHandler() http.Handler`
Reflects on `routes` to build OpenAPI 3.0 JSON. Served at `/openapi.json`.
Uses `reflect` on `ReqType`/`RespType` to generate schemas. No external dependency.

**`apits.go`** — `APITSHandler() http.Handler`
Generates TypeScript client from `routes`. Served at `/openapi.ts`.
One async function per route, typed request/response, uses `fetch`.

**`health.go`** — `HealthHandler(db *sql.DB) http.Handler`
```json
GET /api/health → {"status": "ok", "db": "ok"} or {"status": "error", "db": "error: ..."}
```

**`server.go`** — `Serve(port string, db *sql.DB) error`
```
Builds http.ServeMux:
  /openapi.json  → OpenAPIHandler()
  /openapi.ts    → APITSHandler()
  /api/health    → HealthHandler(db)
  + all routes from Routes()
Calls http.ListenAndServe(port, mux)
```

### Task 6 — `atlas-base/go/cli/cmd/codegen/` — codegen commands

**`codegen.go`** — parent `atlas codegen` command + `SubcommandProvider`

**`sqlc.go`** — `atlas codegen sqlc`
1. Verify `db/queries/` and `db/migrations/` exist in working directory
2. Generate `db/sqlc.yaml` from embedded normative template (substituting Go package name from `go.mod`)
3. Run `sqlc generate` with that config
4. Print generated files

**`openapi.go`** — `atlas codegen openapi <url> [--output <path>]`
1. GET `<url>/openapi.ts`
2. Write to `--output` (default: `./frontend/src/api.gen.ts`)
3. Print success + path written

Both are also mounted in the example's CLI (`atlas.local/example/cli`).

### Task 7 — `atlas-base/go/cli/cmd/compose/` — PROJECT_NAME injection

**`resolve.go`**: add `Project string` to `AtlasExtension`.

**`up.go`**: after resolving files, read `x-atlas.project` from the entry compose file (last in resolved list).
Call `os.Setenv("PROJECT_NAME", project)` before any `docker compose` invocation.
If `x-atlas.project` is empty, warn but continue.

```go
    api.Register(api.Route{Method: "PUT",    Path: "/api/todos/{id}", Summary: "Update todo", Handler: updateTodo, ReqType: &UpdateTodoRequest{}})
```

**`backend/main.go`**:
```go
//go:embed migrations/*.sql
var migrations embed.FS

func main() {
    basedb.RegisterMigrations(basedb.MigrationSet{Prefix: "todo", Files: migrations})
    conn, err := basedb.Open(os.Getenv("DATABASE_URL"))
    if err != nil { log.Fatal(err) }
```

```sql
CREATE TABLE todos (

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**`backend/db/queries/todos.sql`**:

```sql
-- name: ListTodos :many

-- name: CreateTodo :one

-- name: UpdateTodo :one
UPDATE todos SET text = $2, done = $3 WHERE id = $1 RETURNING *;

-- name: DeleteTodo :exec
DELETE FROM todos WHERE id = $1;
```

## Key Design Decisions

- **Always proxy**: nginx never serves static files directly. `FRONTEND_URL` points to the frontend container (prod) or Vite dev server (dev). No `FRONTEND_MODE` switch.
- **Prefix-namespaced migrations**: each project registers its own `MigrationSet` with a unique prefix. Versions count from `0001` per prefix. No collision possible.
- **Package-level route registration**: project routes live in `backend/api/*.go`, registered via `init()` calling `api.Register()`. `main.go` blank-imports the package.
- **Committed generated files**: `query.gen.go` and `api.gen.ts` are committed like source code. Refreshed via `atlas codegen sqlc` and `atlas codegen openapi`.
- **`atlas codegen sqlc`**: generates `db/sqlc.yaml` from an embedded normative template, then runs `sqlc generate`. Projects never write `sqlc.yaml` by hand.
- **Relative compose dependencies**: projects reference framework compose files by relative path in `depends-on-compose`. `atlas compose up` resolves and injects `PROJECT_NAME` from `x-atlas.project`.
- **Shared network**: `postgres.yml` creates `${PROJECT_NAME}-net`; `nginx.yml` joins it as `external: true`. All project services join the same named network.
- **Example is separate go.work**: `atlas-base/example/go.work` is not added to the root `go.work`.
